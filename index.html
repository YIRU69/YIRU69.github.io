硬件工程师学习方向及内容浅谈

发表于 2019-03-05 | 更新于 2019-03-06 | 分类于 硬件工程师知识

序

​ 在满25周岁之际，才有幡然醒悟之感。开始自律，开始思考，开始对自己的人生负责，也许不早，但愿不晚。

我为什么选择了硬件工程师？

​ 我大学读的专业是通信工程，但是上大学选择这个专业原因有2点

  1. 软件学的人多

  2. 硬件比软件难

我是一个喜欢挑战自己的人，当时基于这2点原由，我义无反顾的选择了硬件。一直到现在，在外面听到很多人的经验之谈，普遍都是认为软件的发展比硬件要好，即容易上手，来的收益也快速。也许以前我会对自己的决定有怀疑，但是现在我想明白，做技术这一行，做到头来硬件和软件都是差不多的。以人类的发展来说，做到极致都说优秀的。

我为什么要写这片文章？

​ 最重要的出发点就是为了记录，把自己从幡然醒悟后开始奋斗的经历记录下来，我闲暇会想我成功来回首的时候，那种感觉应该会很甜吧！

​ 再者就是为了慢慢养成好的习惯，做笔记，做记录，多回顾。有些基础知识需要自己不断的巩固，养成一个做笔记的习惯，把我自己当时的学习状态和理解记录下来，回头再看会有用的。

这篇文章写了什么

​

​ 这篇文章主要是我通过在网上搜索一些文章，来对自己选择的硬件工程之路有一个大致的了解，同时对自己的学习方向有一定的规划。

硬件工程师是什么？

​ 百度百科给出的定义是：

硬件工程师Hardware Engineer职位 要求熟悉计算机市场行情；制定计算机组装计划；能够选购组装需要的硬件设备，并能合理配置、安装计算机和外围设备；安装和配置计算机软件系统；保养硬件和外围设备；清晰描述出现的计算机软硬件故障。

下面是在网上自己总结的：

硬件主要是物理实现，软件主要是算法与应用，两者是相互依存的。

硬件的学习成本高，要提升的关键是实践。任何电路设计出来就要做出实物来验证，然后一步一步调试。

学硬件最终是要学习软件的，单纯走硬件只是固步自封。

硬件目前的发展速度对比软件要缓慢，对应的知识更新速度相对减慢。所以很多硬件有经验之谈。

​ 总的来说，硬件是直接产品本身相关的，换句话说，硬件做的就是产品。硬件工程的职位在社会分工繁多杂乱，主要是因为硬件工程师是针对产品的研制，而产品的研制过程中所需要某类型的工程师由产品决定的。比如，做FPGA就需要FPGA工程师，做医疗器件就需要EMC硬件工程师等等。这就导致了，硬件工程师的内容杂而多，同时由于学习成本的原因，很多硬件工程师只是懂一部分硬件内容。

硬件工程师要学什么？

基本知识

​ 产品是由器件和电路组成，那么硬件工程师的学习方向就离不开器件和电路。硬件工程师所需要的最基本知识就在器件和电路中。

  1. 分立器件

    1. 电阻：功耗和容值

    2. 电容

      1. 瓷介电容（CT）：廉价，容量小，主要用于低频电路
      2. 涤纶电容（CL）
      3. 独石电容（CC）：很贵，容量小，性质稳定。一般用于高频电路
      4. 电解电容（CD）：容量大，有正负极之分。主要用于电源电路，脉动电路
      5. 云母电容（CY）：耐压（250v-450v），容量小，很贵，体积大。用于通讯机重要部位等

    3. 电感

    4. 磁珠

    5. 二极管

    6. 三极管

    7. MOS管

    8. 变压器

    9. 光耦

    10. 继电器

    11. 连接器

    12. RJ45

    13. 光模块

    14. 防护器件TVS管

    15. 压敏电阻

    16. 放电管

    17. 保险管

    18. 热敏电阻

  2. 电路

    1. 桥式整流电路

    2. 电源滤波器

    3. 信号滤波器

    4. 微分和积分电路

    5. 共射极放大电路

    6. 分压偏置式共射极放大电路

    7. 共集电极放大电路（射极跟随器）

    8. 电路反馈

    9. 二极管稳压电路

    10. 串联稳压电源

    11. 差分放大电路

    12. 场效应管放大电路

    13. 选频（带通）放大电路

    14. 运算放大电路

    15. 差分输入运算放大电路

    16. 电压比较电路

    17. RC振荡电路

    18. LC振荡电路

    19. 石英晶体振荡电路

    20. 功率放大电路

进阶知识

​ 在产品的器件和电路的知识了解的情况下，进一步去了解学习器件与电路组成的电路系统，以及操作这些系统的工具和语言。在完成产品过程中还要考虑维护、测试以及安全性等方面

  1. 系统

    1. 电源
    2. 存储器
    3. 总线
    4. CPU
    5. 复位
    6. 时钟
    7. 逻辑器件
    8. 接口
    9. 芯片

  2. 工具与语言

    1. C语言

    2. verilog

    3. AD、proteus

    4. ise

    5. python

  3. 维护测试安全
    1. EMC、安规
    2. 热设计、降额设计
    3. PCB工艺、布局、可制造、可测试性设计
    4. PoE供电
    5. 测试知识
    6. 测试仪器使用

我该怎么学？

​ 繁多杂乱的知识，按图索骥的学习太过缓慢。初步的一个学习方法如下：

  1. 了解基础知识

  2. 系统思维

  3. 因地适宜

在基础知识的学习上，要做的就是了解。简单来说就是知道这个东西，对原理上不求甚解。

在系统思维的学习上，用自顶而下的思维去学习，多问为什么。

在因地适宜的学习上，从工作出发，不脱离实际，学的知识要经常使用才能固化到大脑中。
